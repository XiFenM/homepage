最近在重读 *Programming Rust*。第二遍读和第一遍读完全不一样——不是书变了，而是我多了一些之前没踩过的坑。

## 借用检查器最先教会我的事

不是所有权，是**认真给一段数据起名字**。当借用检查器抱怨的时候，它通常是在帮我意识到：这段引用真正的归属并不像我以为的那样清楚。

## 接下来想做什么

- 把一个我用 Python 写过的小工具用 Rust 重写一遍
- 不是为了性能，是为了练手
- 也想看看 `tokio` 的取消语义是不是真的像传言里那么微妙

不急，慢慢来。
