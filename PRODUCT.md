# Product

## Register

brand

## Users

所有访客**统一从晚樵庐（简约站）落地**。其中能多看几眼、好奇心强一些的访客有概率发现"勿忘我"彩蛋——这是设计上的有意分层，不是导航分流。

按到访路径与访客类型分层：

1. **技术同行 (technical peers)** — 看过我某篇博客、在 HN/V2EX/即刻看到我的分享而来。他们能识别工程品味、关心思考过程而不是 KPI 数字。**晚樵庐就够了**——他们快速摘取卡片里的技术内容；如果时间充裕，也可能 hover 到那个微露的卷轴。
2. **朋友 / 老朋友 (friends)** — 已经认识我、被某条动态或某次对话带过来的。他们看晚樵庐做"近况速览"——但更可能是**已经知道勿忘我入口的回访者**，直接走 `/map` URL。
3. **缘分到达的陌生人 (strangers, especially curious women in a similar life stage)** — 通过共同兴趣 (天文、迷宫饭、动漫、游戏)、社交平台或某条偶然链接到达。他们没有目的，只是好奇这个人是谁——**这群人是彩蛋视觉路径的最佳目标**：好奇心强、停留时间长、会真的去 hover 那个卷轴。
4. **兴趣同好 (fellow nerds)** — 因为某个具体兴趣 (比如同样喜欢迷宫饭) 而到达。他们容易在勿忘我的对应兴趣 hotspot 待最久——但也可能直接看了晚樵庐的卡片就满足。

**明确不在受众范围内**：HR / 招聘者。本站不为求职服务——HR 直接走 CV + GitHub 主页。晚樵庐的"技术"卡片面向**同行的技术读者**，不是面向"招聘漏斗"。

**使用情境**：通常是深夜独处 / 通勤 / 周末闲逛的私人时间。访客带着轻度好奇而非紧急任务到达——这意味着**站点不能用"快速跳转 / KPI 提示 / CTA"那一套生产力 UI 语言**。

**他们要完成的事**：在 30 秒到 5 分钟之间，决定"这个人值不值得我继续了解"——晚樵庐给前者，勿忘我给后者（如果他们找到了的话）。

## Product Purpose

这是一个**主体 + 彩蛋 (one site, one secret)** 的个人网站：

- **晚樵庐 (Quiet Mode，站点主体)** 服务于"高效摘取"。访客落地即在此——一张暖灯下的书桌，桌上摊着一堆春绿丝带束扎的卡片（5 张分类内容入口）。访客 30 秒内能感受到"这人是谁、专业方向、兴趣偏好"，靠风格与卡片内容自己说话。**结构本身不偏离大众个人站的常规——能看的人随时离开，能多看的人会发现彩蛋**。

- **勿忘我 (Map Mode，隐藏的彩蛋)** 服务于"慢速探索"。一张精绘等距异世界地图 + 8–10 个 hotspot——但**它不在主导航里**。它是晚樵庐书桌一角微露的旧卷轴，鼠标接近时缓缓展开（V2）；点击进入后视野展开为一整张大地图（S6）；点击观星塔 hotspot 时镜头抬起、羊皮淡化为星空（V3）。访客像在桌游夜里偶然翻到一张老地图一样，逐站揭开"我是个什么样的人"。

**彩蛋触发——三层冗余**：
1. **视觉主路径**：书桌左下/抽屉一角微露的小卷轴。鼠标接近时 V2 慢慢展开，点击进入。
2. **URL 直达**：`/map` 或 `/勿忘我` 直接进——朋友间分享/老访客快速回访用。
3. **页脚冗余**：footer 一句模糊话术（"还在某处游荡 / still wandering somewhere..."）链接到地图。

任意一条都能找到，但只有第 1 条才有"无意中发现"的惊喜。

四类内容统一覆盖：

1. **技术 (Technical)** — 技术博客、工作学习中的想法、对工程文化的观察。简约站直接承载（卡片之一）；勿忘我地图上的「钢铁书库」hotspot 提供另一种入口。**面向技术同行而非招聘方**——可以谈品味、谈选择、谈失败，不必谈 KPI。
2. **生活 (Life)** — 旅行、健身、日常心得感悟。简约站卡片承载；勿忘我「鸢尾平原」「烹饪窝棚」hotspot 提供散点入口。
3. **兴趣 (Interests)** — 游戏、动漫、天文宇宙 / 科幻探索 (这是核心 nerd 锚点)。简约站卡片承载；勿忘我「观星塔」hotspot 触发星空 sub-zone（V3 + S7）。
4. **自我 (Self)** — **仅在勿忘我地图上作为「旅人客栈」hotspot 存在** (恋爱观、想找的人、不想要的人) — 半显性、克制、不广告化。**简约站不出现这条**——只有走到勿忘我并主动翻到客栈的访客才会读到。

**成功标准**：

- 简约站：访客 30 秒内能用一句话描述"这人是谁、做什么、喜欢什么"——**且这句话不来自任何"About Me"段落**，全部来自风格、配色、卡片内容选择本身。同时，**离开率 ≤45%**（接近大众个人站基准）。
- 勿忘我：触达率不强求（这是彩蛋），但触达后愿意停留 ≥5 分钟，主动点开 ≥3 个 hotspot，且至少在一个 hotspot 上停留超过 1 分钟阅读。
- 彩蛋发现：在桌面一角的微露卷轴上 hover ≥1 秒的访客比例 ≥15%（视觉主路径有效性指标）。

## Brand Personality

三个核心词加两个修饰词：**真诚 (sincere)、松弛 (unguarded)、好奇 (curious)** — 修饰：**冷静 (calm)、向往未知 (drawn to the unknown)**。

**声音 (Voice)**：像深夜跟朋友散步聊到自己感兴趣的事情时的语气——不刻意、不卖弄，但聊到喜欢的东西时会眼睛亮一下。**不是导师腔、不是凡尔赛腔、不是网红腔、不是 LinkedIn 腔**。

**调性 (Tone)**：慢一拍。内容密集时不堆砌信息密度，留白让访客喘气；视觉热闹时保持文字克制，反之亦然。整体氛围像桌游夜里的暖灯——不喧闹、不焦虑、不卖弄。

**情绪目标**：

- 晚樵庐（简约站）：访客感觉"哎，这人有点意思"——不是惊艳，是温和的好奇。
- 勿忘我（地图彩蛋）：访客感觉"原来这里还有一层；想跟着这个人在他的地图上走一走"——像 D&D 桌游夜里偶然翻到的同一张地图。
- 彩蛋发现的瞬间：访客感觉"咦——这卷轴还能展开"——不是 UI 引导，是无声的邀请被接住。
- 整体：夜深、灯暖、私下闲谈——而非舞台聚光灯。

## Anti-references

1. **黑底 + 终端字体 + ASCII art 的"hacker chic"模板。** 程序员个人页的最大反例。它把"我是程序员"演成"我在扮演一个程序员"。本站走相反方向：我**恰好**是程序员，但这件事不该在外观上被反复强调。
2. **Linktree-style 按钮堆叠。** 把"一个人"压缩成"一组链接"。本站任何位置都禁止用按钮列作为主信息载体。
3. **about / projects / contact 三段式 SaaS 模板。** AI 训练数据反射。把 "I'm a [role] who builds [stuff]. Here are some projects." 这种公式化结构作为绝对反例。
4. **GitHub contribution grid / Wakatime 时长 / 各种 stats 徽章。** 用"勤奋指标"替代"技术品味"。本站绝不出现。
5. **米哈游 / 二游通用 KV 模板** (发光圆形背板 + 居中角色 + 渐变光晕)。即便勿忘我地图走迷宫饭主调，也不能滑向这种二游 KV。
6. **专门的"About Me"段落 / 30 秒人设介绍式开场白** ("Hi, I'm [name], a [role] passionate about [things]")。教科书级反例。本站让风格 + 元素 + 内容选择本身回答"这人是谁"，而非用说明文。
7. **凡尔赛腔与卖弄腔。** 项目介绍里"曾在 X 公司主导 Y 系统的 Z 优化，带来 N% 提升"这种 LinkedIn 简历复制粘贴语气与本站冲突。
8. **"找伴侣"广告语 / 交友站措辞。** 即便存在勿忘我「旅人客栈」hotspot，也禁止"我在找一个 [adjectives] 的人"这种征友体——回到内容描述 ("我相信什么、我感动于什么、我害怕什么")，让读者自己判断。
9. **彩蛋的过度声张。** 任何"点这里看彩蛋 / Click here for surprise / 探索更多 →" 这类显式 CTA 都违反勿忘我的"无声邀请"语义。彩蛋的存在感来自视觉本身（卷轴一角）和访客好奇心，不来自文字提示。
10. **"双站对等"残留。** 旧版 spec 把简约站与探索站设为同等公民、首页是 fork。本架构已**废弃**该模式：晚樵庐就是站点本身，勿忘我是它的一个秘密，不是它的另一半。任何把两者并列展示的导航 / 视觉对位都视为反例。

## Design Principles

1. **主体 + 彩蛋 (one site, one secret)。** 站点结构是"晚樵庐 = 简约主站；勿忘我 = 隐藏地图"。两者**不对等**：晚樵庐是访客落地的全部，勿忘我是它的一个秘密。**禁止把它们做成同等公民式的导航**（旧版 fork 架构已废弃）；**禁止用文字 CTA 引导访客去发现彩蛋**。
2. **风格自我介绍 (the design IS the about-me)。** 简约站没有 "关于我" 段落。访客 30 秒内对"这人是谁、做什么、喜欢什么"的判断必须靠**站点风格 + 置顶内容选择 + 微动效**完成。**禁止在任何位置出现传统 self-intro 文字块**。
3. **彩蛋的"无声邀请" (the easter egg invites quietly)。** 勿忘我的发现路径分三层（视觉 / URL / footer），但**任何一层都禁止使用解释性文字、提示气泡、引导动画**。卷轴一角微露 + V2 慢慢展开本身是邀请；它的语义是"如果你看到了，就请进来；如果没看到，本来也不强求"。
4. **四源继承，非平分 (inherit, don't average)。** 站点风格脉络明确：
   - 勿忘我主调 = 迷宫饭 (羊皮卷 + 桌游夜)
   - 勿忘我「观星塔」sub-zone = 星语共愿 (深夜天文台 + 手写笔触)
   - 晚樵庐（简约站）主调 = 罗小黑联动 (春绿 + 纸感 + 档案式)
   - 全站签名 / 章节落款 = 怀黍离 (印章红 + 卷轴留白)
   任何元素必须能追溯到上述四源之一，不能出现"四不像的折中"。
5. **静态画 + 微动效 > 拖拽交互 (motion in stillness)。** 勿忘我地图不做手动拖拽。用一张精绘等距 (isometric) 异世界地图 + 8–10 个 hotspot + 鼠标接近时的微动效——靠**画面与动效本身的精度**赢得惊艳感，而不是靠交互复杂度。
6. **AI 生美术，人写提示词 (AI for assets, prompts written by hand)。** 大量视觉资源 (异世界地图、hotspot 插画、宇宙星图) 交给 AI 图像生成。设计文档对每一张"AI 资产"必须提供：用途 + 必备视觉特征 + 风格锚点 + 反例。**不写"看着办"的提示词**。
7. **半显性的"自我"段 (one honest hotspot, no marketing)。** 勿忘我地图上有且仅有一个「旅人客栈」hotspot 承载"About Me as a Person"——克制、真诚、不写交友广告语。**禁止在晚樵庐（简约站）任何位置出现"找伴侣"类信号**——简约站访客多为技术同行 / 朋友 / 路人，不该被强加这条；只有走到勿忘我并主动翻到客栈的访客才会读到。
8. **中英对等，中文默认 (bilingual peers, Chinese default)。** 所有内容 / 菜单 / 章节标题 / 错误状态都有中英两版，字号比与排版比一致——不是一方做小。语言切换按钮在所有页面同位置可达，切换不丢失访客当前位置 (在 hotspot X 处切换语言后仍在 hotspot X)。**禁止机翻**——英版需由人 / 高质 LLM 翻译并保留语气。

## Accessibility & Inclusion

- **WCAG**：全站文本对比度 ≥ AA 级 (4.5:1)。晚樵庐基础色板 (春绿 + 纸感) 在白底上对比度容易掉到 3:1——所有功能性文本必须落在 ink-black 上、而非春绿块上。
- **prefers-reduced-motion**：勿忘我地图 hotspot 的呼吸 / 抖动 / 光晕、地图上的尘粒 / 风 / 星点等微动效全部依赖动效——必须为动晕用户提供完整静态版本，hotspot 仅显示静态发光环。**晚樵庐桌面环境循环（V1）和卷轴 hover 展开（V2）也必须有 reduced-motion 静态等效**——V1 替换为 S3 静态背景，V2 替换为静态地图缩略图直接显示。
- **prefers-color-scheme**：勿忘我地图底色羊皮卷 + 观星塔深蓝为深夜浏览设计；晚樵庐为浅色环境设计——两者尊重系统主题偏好，但不强制 dark-mode 切换 (因为色板与情绪绑定，强制反转会破坏调性)。
- **色觉**：春绿 + 灰墨在二色觉用户处区分度不足——晚樵庐的 link / hover / visited 状态除颜色外必须叠加 underline / weight 双通道反馈。
- **键盘**：勿忘我地图 hotspot 必须可 Tab 导航，顺序遵循"叙事推荐顺序"而非地理坐标——先到的 hotspot 应该是最适合作为"入口故事"的那一个。**彩蛋触发也必须有键盘等效**——卷轴 ROI 在键盘 Tab 流中可达，按 Enter 等同于点击进入勿忘我。
- **彩蛋的可达性兜底**：彩蛋视觉路径对**部分访客天然不可达**（reduced-motion 用户、屏阅器用户、移动端小屏可能看不清卷轴一角）。这就是为什么必须保留 `/map` URL + footer 链接两条冗余路径——它们对所有访客可见、可点击、可分享。
- **隐性受众保护**：勿忘我「旅人客栈」hotspot 的内容必须克制具体——回避"我在找一个 [adjectives] 的人"类语句，改为"我相信 / 我感动于 / 我害怕的事"等内容描述。让读者基于"我们是不是同类人"自行判断，而不是被强行投放。
